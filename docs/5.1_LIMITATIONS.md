# 5.1. Keterbatasan Sistem

## Keterbatasan Teknis

Sistem beroperasi dengan arsitektur Node.js single-threaded, membatasi kemampuan pemrosesan bersamaan.

Database SQLite membatasi operasi write hanya untuk satu writer bersamaan, mempengaruhi skenario high-concurrency.

Tidak ada layer caching built-in; semua query langsung mengakses database tanpa optimasi.

Integrasi WhatsApp bergantung pada library whatsapp-web.js yang unofficial, menciptakan risiko dependensi.

Hanya basic in-memory caching yang diimplementasi untuk trigger counts dengan TTL 5 menit.

Sistem hanya mendukung database Oracle, MySQL, dan SQLite; PostgreSQL tidak didukung meskipun dependency ada.

Tidak ada kemampuan horizontal scaling; arsitektur single server membatasi potensi pertumbuhan.

Pemrosesan synchronous tanpa message queuing dapat menyebabkan operasi blocking.

Tidak ada dukungan container orchestration (Docker/Kubernetes) untuk otomasi deployment.

Tidak ada pipeline CI/CD; proses deployment manual meningkatkan risiko error.

Hanya basic logging saja; tidak ada monitoring advanced dengan Prometheus, Grafana, atau ELK stack.

Tidak ada framework testing otomatis; hanya testing manual.

Tidak ada interface dokumentasi API (Swagger/OpenAPI); diperlukan discovery API manual.

## Keterbatasan Operasional

Sistem memerlukan koneksi internet stabil; operasi offline tidak didukung.

Integrasi WhatsApp memerlukan re-authentication berkala menggunakan pemindaian QR code.

Hanya satu sesi WhatsApp aktif yang didukung per instance.

Throughput pesan dibatasi oleh kendala protokol WhatsApp Web (sekitar 500 pesan per menit).

Interface web dioptimalkan untuk browser desktop (Chrome, Edge); browser mobile tidak sepenuhnya didukung.

Tidak ada analytics advanced atau reporting otomatis; hanya statistik basic yang tersedia.

Tidak ada sistem notifikasi email atau SMS; notifikasi WhatsApp saja.

Sistem role user basic dengan distingsi admin/user sederhana.

Tidak ada kemampuan export data ke CSV, Excel, atau format lain.

Tidak ada fungsionalitas backup dan restore otomatis.

Tidak ada dukungan interface multi-bahasa.

Pengiriman notifikasi bergantung pada konektivitas WhatsApp dan ketersediaan device.

Backup dan recovery data harus dilakukan secara manual oleh administrator sistem.

## Kendala Legal & Compliance

Semua data user disimpan secara lokal di server perusahaan dan harus mematuhi internal Data Protection Policy.

Sistem mengikuti praktik keamanan berbasis ISO 27001 untuk authentication dan audit logging.

Tidak ada data user personal yang dikirim ke third-party AI providers tanpa consent eksplisit.

Sistem dual authentication (JWT untuk web users, API Key untuk bots) menyediakan keamanan enterprise-grade.

Compliance OWASP Top 10 baik dengan authentication, input validation, dan security headers yang proper.

Compliance GDPR dan PDPA partial; missing advanced encryption dan fitur data deletion.

Compliance WhatsApp Business API non-compliant karena penggunaan unofficial API.

Keamanan sistem kontrol industrial mengikuti standar ISA/IEC 62443 level basic.

Compliance standar sistem SCADA minimal.

Compliance NIST Cybersecurity Framework moderate dengan kontrol basic yang diimplementasi.

Policy retention audit log 90 hari untuk compliance data retention.

Basic role-based access control tanpa fitur RBAC advanced.

Prosedur incident response minimal yang saat ini diimplementasi.

## üîê **5.2. SECURITY PROCEDURES**

### **Specific Security Procedures Required**

**Beyond standard software development practices (OWASP Top 10, secure coding), the WA-Integration system requires implementation of specific security procedures due to its industrial integration and dual authentication architecture:**

#### **5.2.1 Industrial Control Systems (ICS) Security Procedures**

**Network Segmentation Requirements:**
- **Mandatory Implementation:** Physical/logical separation between IT network and industrial control network
- **Technical Requirements:**
  - DMZ implementation for AVEVA PI integration
  - Firewall rules for industrial protocol filtering
  - VLAN segregation for SCADA components
- **Compliance:** ISA/IEC 62443 Level 2 minimum
- **Current Gap:** No network segmentation implemented

**Industrial Data Protection:**
- **Data Classification:** Implement industrial data classification (public, internal, confidential, restricted)
- **Encryption Requirements:** AES-256 encryption for industrial data at rest and in transit
- **Access Controls:** Role-based access control for industrial data with principle of least privilege
- **Audit Requirements:** Comprehensive audit logging for all industrial data access

#### **5.2.2 Dual Authentication System Security Procedures**

**API Key Management:**
- **Key Rotation:** Automated API key rotation every 90 days
- **Key Storage:** Secure key storage using Hardware Security Modules (HSM) or equivalent
- **Access Monitoring:** Real-time monitoring of API key usage patterns
- **Revocation Procedures:** Immediate key revocation procedures for compromised keys

**JWT Token Security:**
- **Token Encryption:** Implement JWE (JSON Web Encryption) for sensitive JWT payloads
- **Short-lived Tokens:** Maximum token lifetime of 15 minutes for high-security operations
- **Token Blacklisting:** Implement token blacklist for compromised sessions
- **Refresh Token Security:** Secure refresh token handling with rotation

#### **5.2.3 WhatsApp Integration Security Procedures**

**Unofficial API Risk Mitigation:**
- **Traffic Analysis:** Implement traffic pattern analysis to detect API changes
- **Fallback Procedures:** Develop fallback notification mechanisms (email/SMS)
- **Rate Limiting:** Implement comprehensive rate limiting to prevent API abuse
- **Session Security:** Secure WhatsApp session storage and recovery procedures

**Business API Migration:**
- **Migration Planning:** Develop migration plan to official WhatsApp Business API
- **Cost Analysis:** Budget planning for WhatsApp Business API licensing
- **Testing Procedures:** Comprehensive testing of new API integration
- **Rollback Plan:** Business continuity plan during API migration

#### **5.2.4 Database Security Procedures**

**Multi-Database Security:**
- **Connection Security:** Encrypted connections for all database types (Oracle, MySQL, SQLite)
- **Query Auditing:** Implement query logging and analysis for security monitoring
- **Backup Security:** Encrypted database backups with integrity verification
- **Access Controls:** Database-level access controls beyond application layer

**SQLite Security Limitations:**
- **File System Security:** Secure SQLite file permissions and access controls

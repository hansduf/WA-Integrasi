Integration with External Systems — Short reference

- AVEVA PI — Sistem telemetry/historian; dipanggil lewat backend untuk query real-time/historis.
- Oracle Database — Sumber data operasional (dibaca oleh backend).
- MySQL — Sumber data tambahan dan target migrasi (dibaca oleh backend; migrasi ada di `archive/`).
- WhatsApp Web Service — Bot menggunakan `whatsapp-web.js` + Puppeteer; status/session persisted locally.
- AI API Endpoint — Internal AI triggers & chat endpoints; digunakan oleh bot/backend.
- Backend API (Express) — Orkestrator/proxy untuk semua integrasi; menyimpan konfigurasi data_sources di SQLite.
- Local SQLite — Penyimpanan pusat untuk konfigurasi (`data_sources`) dan state ringan.
- Archive scripts — One-off scripts untuk migrasi/tes; berisi contoh BASE_URL/API key (HARUS direview sebelum dipakai).
- HTTP client (axios) — Dipakai di backend & bot; sebaiknya distandarisasi (timeouts, retries, headers dari env).

Notes:
- Semua connection strings/credential untuk sumber eksternal disimpan PUSAT di SQLite (`data/app.db` → tabel `data_sources`) — jangan simpan kredensial di frontend atau commit ke repo.
- Archive scripts sering mengandung contoh literal (`universal-api-key-2025`) — redact/replace sebelum menjalankan.

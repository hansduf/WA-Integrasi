# 5.3 Usability & Accessibility

1. User-Friendly Interface
• Antarmuka dirancang dengan komponen yang jelas dan konsisten, memudahkan orientasi pengguna.
• Kontrol utama memiliki label yang sesuai dan fokus visual yang mendukung navigasi keyboard.

2. Accessibility Standards
• Praktik aksesibilitas dasar telah diterapkan: elemen semantik, atribut ARIA pada komponen penting, dan teks alternatif pada gambar.
• Kepatuhan formal terhadap WCAG (A/AA/AAA) belum disahkan; diperlukan audit formal untuk klaim kepatuhan.

3. Language & Localization
• Format lokal untuk tanggal/waktu digunakan di area laporan.
• Sistem belum memiliki mekanisme i18n terpusat; perlu perencanaan jika akan mendukung multi-bahasa.

4. Cross-Platform Usability
• Desain responsif mendukung berbagai ukuran layar; interaksi dasar tersedia untuk desktop dan perangkat sentuh.
• Disarankan melakukan pengujian manual pada perangkat nyata (mis. screen reader mobile, keyboard-only navigation) untuk verifikasi penuh.

5. Training & Documentation
• Dokumentasi khusus aksesibilitas untuk pengguna akhir belum tersedia.
• Disarankan menambahkan panduan singkat untuk pengguna dan administrator tentang fitur aksesibilitas, navigasi keyboard, dan troubleshooting dasar.

Kesimpulan singkat
• Aplikasi menerapkan praktik aksesibilitas dasar yang baik dan mendukung kebanyakan kebutuhan usability.
• Untuk menyatakan kepatuhan WCAG secara formal diperlukan audit aksesibilitas dan beberapa perbaikan lanjutan (mis. pemeriksaan kontras warna, focus management, dan i18n).

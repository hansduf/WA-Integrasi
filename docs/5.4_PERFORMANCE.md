## 5.4 Performance

Dokumen ini mencatat temuan performa yang teridentifikasi dari kode sumber dan artefak konfigurasi. Isi bersifat faktual dan hanya merekam apa yang saat ini diimplementasikan.

1. System Load
- Komponen utama: layanan belakang (server aplikasi), antarmuka pengguna (aplikasi web), dan layanan WhatsApp (bot). Bot cenderung menimbulkan beban terbesar karena setiap sesi bot menjalankan sebuah browser kecil di latar belakang yang menggunakan CPU dan memori.
- Database saat ini menyimpan data ke file lokal; ini cocok untuk jumlah bacaan ringan dan sedikit penulisan, tetapi memiliki batasan jika banyak penulisan terjadi bersamaan.

2. Processing Performance
- Layanan belakang: server aplikasi menyediakan API yang dipakai oleh antarmuka dan bot untuk membaca/menyimpan data.
- Antarmuka pengguna: halaman web menggunakan teknik pemecahan daftar (mis. halaman) agar tidak memuat terlalu banyak data sekaligus. Beberapa bagian klien mencoba kembali secara otomatis saat menunggu layanan siap.
- Bot (WhatsApp): bot berjalan sebagai program terpisah yang membuka browser kecil di latar belakang untuk berinteraksi dengan WhatsApp. Bot mengambil pengaturan dari server, menyimpan status (termasuk gambar QR saat perlu), dan menulis file status kecil di server. 3. Waktu Respon (Target)
- Dokumen ini belum menetapkan target waktu respon resmi karena tidak ada data operasi yang tercatat. Kode tidak menulis metrik waktu, sehingga angka spesifik tidak tersedia.
- Permintaan baca ke server biasanya cepat dalam kondisi normal, tetapi dokumen tidak berisi angka terukur.
- Penulisan data ke file lokal dapat melambat jika banyak penulisan terjadi bersamaan.
- Respon bot terhadap pesan dapat melambat bila banyak sesi bot berjalan bersamaan karena setiap sesi menjalankan browser kecil.

4. Scalability
- Skalabilitas saat ini terbatas pada level proses: tidak ditemukan mekanisme skala otomatis atau orkestrasi pada kode. Pembatasan utama:
	- Database file-based membatasi skala paralel untuk operasi tulis.
	- Bot berbasis headless browser membutuhkan resource per sesi, sehingga jumlah sesi simultan dibatasi oleh kapasitas host.

5. Monitoring & Optimization (apa yang tersedia sekarang)
- Tersedia audit/logging yang mencatat operasi backend dan log error untuk bot; artefak session cache untuk bot juga ada.
- Tidak ditemukan integrasi metrik terpusat atau RUM pada frontend. Observability terbatas pada log file dan artefak lokal.

Catatan penutup
- Laporan ini hanya mendokumentasikan kondisi yang ada saat ini. Jika diperlukan, dokumen dapat diperluas dengan: a) estimasi angka beban (MAU, RPS, pesan/bulan), b) target respons terukur (SLO), dan c) checklist monitoring yang direkomendasikan. Untuk perluasan tersebut, mohon konfirmasi angka operasional yang diinginkan.

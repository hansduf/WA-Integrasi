# 5.2 PROSEDUR KEAMANAN

## 1. Authentication & Authorization

•	Sistem mengimplementasikan sistem autentikasi dual yang mendukung akses berbasis web dan operasi bot otomatis.
•	User web melakukan autentikasi menggunakan JWT (JSON Web Tokens) dengan manajemen sesi yang aman.
•	Operasi bot WhatsApp menggunakan autentikasi API key.
•	Sesi user dikelola dengan waktu kadaluarsa token yang sesuai (15 menit untuk access token, 7 hari untuk refresh token).
•	Sistem menyertakan fitur keamanan akun komprehensif seperti monitoring upaya login, mekanisme penguncian akun, dan hashing password yang aman menggunakan bcrypt dengan salt rounds yang memadai.

## 2. Data Protection

•	Semua data sensitif, termasuk kredensial user dan data sistem AVEVA PI, disimpan dengan aman dalam aplikasi.
•	Password dienkripsi menggunakan algoritma hashing bcrypt standar industri.
•	Transmisi data antara sistem dan layanan eksternal (WhatsApp API, sistem AVEVA PI) dilindungi melalui protokol enkripsi HTTPS/TLS.
•	Sistem mengimplementasikan validasi input komprehensif menggunakan Joi schemas untuk mencegah serangan injection dan input data berbahaya.
•	Rate limiting diterapkan di berbagai endpoint untuk mencegah serangan brute force dan penyalahgunaan sistem.

## 3. Database Security

•	Sistem menggunakan database SQLite dengan prepared statements untuk semua operasi database, memastikan perlindungan terhadap serangan SQL injection.
•	Koneksi database dikelola secara efisien dengan penanganan transaksi yang proper dan constraint foreign key.
•	Database menyertakan kemampuan audit logging komprehensif, menyimpan aktivitas user, timestamp, alamat IP, dan aksi yang dilakukan.
•	File database dibuat secara otomatis di direktori data khusus dengan kontrol akses yang sesuai.

## 4. System Monitoring & Logging

•	Audit logging komprehensif diimplementasikan di semua operasi sistem kritis termasuk autentikasi user, modifikasi data, sinkronisasi AVEVA PI, dan maintenance sistem.
•	Log mencatat informasi penting seperti user ID, timestamp, alamat IP, user agent, dan aksi spesifik yang dilakukan.
•	Sistem menyediakan monitoring keamanan real-time melalui endpoint security overview khusus yang melacak sesi aktif, upaya login gagal, dan aktivitas mencurigakan.
•	Event keamanan dikategorikan dan dicatat untuk keperluan compliance dan forensik.

## 5. Compliance with Standards

•	Sistem mematuhi praktik keamanan standar industri untuk aplikasi Node.js, termasuk sanitasi input yang proper, validasi content-type, dan implementasi middleware keamanan.
•	CORS (Cross-Origin Resource Sharing) dikonfigurasi untuk membatasi akses ke origin yang diotorisasi saja.
•	Manajemen sesi mengikuti praktik aman untuk mencegah session hijacking dan akses tidak sah.
•	Aplikasi mengimplementasikan multiple layer security middleware termasuk helmet untuk security headers, rate limiting, dan validasi request.

## 6. Security Audit Requirement

•	Sistem menggabungkan prosedur keamanan yang melebihi praktik standar development software, termasuk mekanisme autentikasi dual, audit trails komprehensif, dan monitoring keamanan real-time.
•	Meskipun langkah-langkah keamanan yang diimplementasikan memberikan perlindungan yang kuat, organisasi memerlukan supplier potensial untuk menjalani audit keamanan tambahan guna memastikan kepatuhan dengan standar keamanan organisasi dan persyaratan regulasi.
•	Audit ini akan memverifikasi postur keamanan, memvalidasi implementasi kontrol keamanan, dan mengkonfirmasi kepatuhan terhadap kebijakan perlindungan data sebelum deployment production.

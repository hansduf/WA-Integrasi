## 6.2 Technological requirements

Dokumen ini menjawab pertanyaan operasional tentang kebutuhan teknologi dan infrastruktur untuk menjalankan aplikasi, berdasar temuan faktual dari kode dan dokumen dalam repositori. Teks ditulis ringkas dan faktual.

1) Perangkat yang harus didukung
- Aplikasi sisi server (backend dan bot) dirancang untuk dijalankan pada mesin server yang dapat menginstal Node.js dan menjalankan proses latar (background processes). Bot WhatsApp menggunakan Puppeteer untuk menjalankan browser headless; oleh karena itu host untuk bot harus mendukung eksekusi proses browser headless yang diperlukan oleh Puppeteer.
- Antarmuka web (frontend) bersifat responsif dan ditujukan agar dapat diakses dari perangkat desktop dan perangkat seluler (lihat penggunaan kelas responsif Tailwind di frontend).

2) Sistem operasi yang didukung
- Kode bot menyertakan penanganan proses khusus untuk Windows (taskkill) dan untuk sistem Unix-like (pkill), sehingga kode menunjukkan dukungan operasional pada Windows dan Linux/Unix.
- Backend dan frontend berbasis Node.js — secara umum dapat dijalankan pada Linux, macOS atau Windows yang mendukung versi Node.js yang sesuai.

3) Resolusi layar / tampilan
- Frontend menggunakan pendekatan responsif (Tailwind CSS, grid responsif dan kelas ukuran teks). Tidak ada batasan resolusi khusus di kode; antarmuka akan menyesuaikan pada berbagai ukuran layar (mobile, tablet, desktop).

4) Apakah software dijalankan di lingkungan yang sudah ada?
- Repositori tidak menyertakan artefak container (tidak ada Dockerfile atau docker-compose.yml) dan tidak ada manifest Kubernetes (dokumen `5.1_LIMITATIONS.md` menyebut tidak ada dukungan container/orchestration). Namun, terdapat dokumentasi dan referensi penggunaan PM2 sebagai process manager (`docs/PHASE5_DAY8_COMPLETED.md`, `docs/PROJECT_COMPLETION_REPORT.md`).
- Implikasi faktual: ada catatan proses manajemen (PM2) tetapi tidak ada skrip otomatisasi infrastruktur atau container yang disertakan.

5) Infrastruktur yang tersedia saat ini (bukti)
- Bukti di repo: dokumen proyek menyebut konfigurasi dan penggunaan PM2 untuk menjalankan proses; tidak ada Docker/Kubernetes manifest.
- Tidak ditemukan file provisioning (mis. Terraform/CloudFormation) atau skrip deployment otomatis yang menjelaskan penyediaan server.

6) Perlukah calon penyedia juga memberikan penawaran infrastruktur?
- Fakta dari kode/dokumen: repositori tidak mencatat adanya infrastruktur yang disediakan atau terotomasi. Jika organisasi Anda belum memiliki server atau lingkungan yang siap menjalankan Node.js dan Chromium headless, maka penyedia perlu mendapat instruksi apakah mencakup penyediaan infrastruktur. Dokumen ini hanya mencatat keadaan repo; keputusan untuk meminta penawaran infrastruktur adalah keputusan bisnis yang harus ditentukan oleh pihak yang membuat tender.

7) Persyaratan infrastruktur (faktual, sebagai daftar kebutuhan minimum operasional yang terlihat dari kode)
- Host untuk backend: mesin yang dapat menjalankan Node.js dan menyimpan file database SQLite (akses file lokal). Kebutuhan disk untuk menyimpan file DB dan session cache.
- Host untuk bot: mesin yang dapat menjalankan proses Node.js dan mengeksekusi Chromium/Chrome headless; akses untuk menjalankan proses sistem (kemampuan menghentikan proses Chromium ketika diperlukan).
- Port dan jaringan: backend API dan frontend perlu terpapar pada port HTTP(S) yang dikonfigurasi (kode menggunakan API lokal `http://localhost:<port>` pada bot selama integrasi). Sistem mengandalkan komunikasi HTTP antara bot dan backend.
- Penyimpanan sesi: direktori sesi (`./sessions`, `.wwebjs_auth`, `.wwebjs_cache`) dan file status (`./.status/whatsapp-status.json`) harus dapat ditulis oleh proses bot.
- Process manager: dokumen proyek menyebut PM2; lingkungan produksi kemungkinan memerlukan process manager untuk menjaga aplikasi berjalan (contoh: PM2 atau sejenis), tetapi repo tidak memaksa satu pilihan.

Referensi singkat
- `5.1_LIMITATIONS.md` — menyebut tidak ada Docker/Kubernetes
- `docs/PHASE5_DAY8_COMPLETED.md`, `docs/PROJECT_COMPLETION_REPORT.md` — referensi PM2
- `wa/index.js` — penanganan proses Chromium (taskkill/pkill) dan direktori sesi

Catatan penutup
- Isi ini hanya mendokumentasikan kebutuhan dan kondisi yang dapat diverifikasi dari kode dan dokumen di repositori. Keputusan operasional (mis. apakah meminta vendor menyertakan penawaran infrastruktur) adalah kebijakan pihak pemilik proyek dan tidak diambil di sini.
